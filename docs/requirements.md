# 需求规格说明书

## 1. 项目背景

摄影机构在日常运营中面临学员档案分散管理、报名缴费流程不透明、作品提交与点评难以追踪等痛点。本系统旨在通过信息化手段整合上述业务流程，提升管理效率。

## 2. 系统角色与权限需求

| 角色 | 角色标识 | 核心权限 |
|------|----------|---------|
| 超级管理员 | admin | 全功能 + 用户管理 + 系统审计 |
| 教务/前台 | staff | 学员档案、课程班级、报名缴费、统计查看 |
| 教师 | teacher | 查看授课班级/学员、点评作品、查看作品进度 |
| 学员 | student | 查看个人报名/课表、提交作品、查看点评、查看提醒 |

## 3. 功能模块需求

### 3.1 用户认证模块
- FR-001：支持账号密码登录
- FR-002：登录失败 5 次锁定账号（防爆破）
- FR-003：新注册用户需管理员审核（pending → approved）
- FR-004：已拒绝（rejected）和待审核（pending）账号无法登录
- FR-005：支持学员注册（默认 student 角色）

### 3.2 学员档案管理
- FR-011：staff/admin 可新增、修改学员档案
- FR-012：学员档案含：姓名、手机号（唯一）、身份证、地址、备注
- FR-013：学员档案可关联系统登录账号（user_id）
- FR-014：学员可查看自己的档案信息
- FR-015：所有档案操作写入审计日志

### 3.3 课程与班级管理
- FR-021：admin/staff 可 CRUD 课程（名称唯一、学费、课时周数、状态）
- FR-022：admin/staff 可 CRUD 班级（编号唯一、关联课程、分配教师、开课日期）
- FR-023：teacher 可查看自己授课的班级信息
- FR-024：可获取教师列表用于班级分配

### 3.4 报名与缴费管理
- FR-031：admin/staff 可为学员创建报名记录（关联学员 + 班级）
- FR-032：报名状态：active（在读）/ pending（待付款）/ withdrawn（退班）/ completed（已完成）
- FR-033：报名含服务周期（service_start / service_end）
- FR-034：admin/staff 可录入缴费流水（定金/尾款/退款）
- FR-035：支持多种付款方式（现金/微信/支付宝/转账）
- FR-036：缴费操作写入审计日志（退款标记更高风险）

### 3.5 作品管理
- FR-041：student 可提交作品（标题、描述、文件URL、截止日期）
- FR-042：teacher/admin 可点评作品（评分 0-100、评语）
- FR-043：作品状态：submitted（已提交）/ reviewed（已点评）/ overdue（已逾期）
- FR-044：作品点评写入审计日志

### 3.6 提醒系统
- FR-051：Dashboard 展示 24 小时内即将截止的作品（黄色警告）
- FR-052：Dashboard 展示已超过截止日期的作品（红色危险）
- FR-053：admin 可见过去 24 小时的系统风险操作统计

### 3.7 审计与风控
- FR-061：所有敏感操作记录 SysLog（操作人、行为、对象、IP、风险等级）
- FR-062：非工作时间（北京时间 0-7 点）操作标记 risk_level=1（警告）
- FR-063：审计日志可按操作人账号、起始日期、风险等级筛选
- FR-064：admin 可查看业务流水审计和系统操作日志

## 4. 非功能需求

- NFR-001：前端所有数据渲染使用 escapeHtml 防 XSS
- NFR-002：所有表单提交需携带 CSRF Token
- NFR-003：数据库使用 SQLAlchemy ORM，防 SQL 注入
- NFR-004：系统支持 MySQL 数据库（可扩展至其他 RDBMS）
- NFR-005：响应时间 ≤ 2 秒（本机环境）
